Robocopy が失敗してしまう事象の調査

目次

  • [A] 事象の再現性がある場合
  • [B] 過去に発生したエラーであり、再現性が無い場合

(本記事はマイクロソフト社員によって公開されております)

こんにちは、日本マイクロソフトの Windows プラットフォーム サポートの三浦です。

本日は Robocopy によるファイル コピーの処理中にエラーが発生した際に採取すべき資料を紹介いたします。まずは調査の初動に有効な以下の 2 パターンをご案内いたしますので、弊社にお問い合わせいただく時には、ぜひご確認ください。

  • [A] 事象の再現性がある場合

    1. 追跡型のトレース ログの採取を開始後に Robocopy を実行する
    2. トレース ログ採取後、サポート情報ログ (イベント ログやシステム情報等) を採取する

  • [B] 過去に発生したエラーであり、再現性が無い場合

    1. サポート情報ログ (イベント ログやシステム情報等) を採取する

[A] 事象の再現性がある場合

  1. 以下の URL より “shacollector.zip” をダウンロードおよび展開いただき、対象端末へ配置します。

    ダウンロード先: shacollector.zip

    shacollector の使い方についてはこちら

    (注意) shacollector は機能の拡張などにより内容が変更されることがあります。 そのため、毎回上記のダウンロード先から shacollector を取得いただき利用してください。

  2. 管理者権限のコマンドプロンプトから配置した “shacollector” フォルダーへ移動して、以下のコマンドを実行の上、Process Monitor ログの採取を実施します。

    shacollector.bat trace procmon start C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  3. コピー元、もしくは、コピー先パスが共有フォルダーやネットワーク ドライブの場合は、下記のコマンドを実行の上、パケット キャプチャの採取を実施します。

    shacollector.bat trace packet start size 2048 C:\mslog

    ※ ローカル ドライブ間でのコピーの場合は本手順は不要です。 ※ 最大ファイルサイズは 2048 以外のサイズ (MB) を size オプションに指定することも可能です。 ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  4. Robocopy を実行し、事象を再現させます。この際、/log: オプションを付与して Robocopy を実行し、実行ログの出力を行ってください。

    ※ Robocopy のオプション詳細は下記の弊社公開情報をご参照ください。 robocopy

  5. 事象の再現後、以下のコマンドを実行してトレース ログを停止します。

    shacollector.bat trace procmon stop

  6. パケット キャプチャの採取を行っている場合は、下記のコマンドを実行してログを停止します。

    shacollector.bat trace packet stop

  7. 最後に以下のコマンドを実行して、サポート情報ログを採取します。

    shacollector.bat support all C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  8. 生成されましたログの出力先フォルダーと、/log: オプションで指定したパスに出力された Robocopy の実行ログを圧縮の上、弊社までご提供ください。

[B] 過去に発生したエラーであり、再現性が無い場合

  1. 以下の URL より “shacollector.zip” をダウンロードおよび展開いただき、対象端末へ配置します。

    ダウンロード先: shacollector.zip

    shacollector の使い方についてはこちら

    (注意) shacollector は機能の拡張などにより内容が変更されることがあります。 そのため、毎回上記のダウンロード先から shacollector を取得いただき利用してください。

  2. 管理者権限のコマンドプロンプトから配置した “shacollector” フォルダーへ移動して、以下のコマンドを実行の上、サポート情報ログの採取を実施します。

    shacollector.bat support all C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  3. 生成されましたログの出力先フォルダーを圧縮の上、弊社までご提供ください。また、/log: オプションで Robocopy の実行ログを出力している場合は、実行ログも併せてご提供ください。

以上です。
