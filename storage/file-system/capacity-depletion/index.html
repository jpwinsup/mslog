意図せず空き容量が枯渇してしまった事象の調査

目次

  • 情報採取手順

これまで弊社サポートのトラブルシューティングに活用されていたログ採取ツール “Shacollector” がリニューアルされます。2023 年 9 月 から、TSS.ps1 を用いた手順に順次変更されて、より包括的なログ採取が可能になります。

(本記事はマイクロソフト社員によって公開されております)

こんにちは、日本マイクロソフトの Windows プラットフォーム サポートです。

本日は意図せずドライブの空き容量が逼迫する問題をお問い合わせいただく際に (もしくはお客様にて調査をご実施いただく際に)、採取いただいていると良い資料採取についてご案内いたします。

このような問題への一般的なアプローチとしては、まずどのディレクトリで多くの容量が使用されているかを特定し、次にそこに (肥大化したログ ファイルなど) 不要なファイルがないかを調べることが有効です。

以下の手順では、そのような調査 (+α) に有効な情報を一括して採取することができます。 弊社にお問い合わせのを検討いただいている場合には、予め採取しておくと迅速に対応することができます。

情報採取手順

※ バッチファイルではログ ファイルのコピーやディレクトリ情報を出力するコマンドを実行しております。 システムに大きな負荷をかけるような操作は実行しませんが、Disk, CPUへ若干の負荷は発生します。 サービスへの影響が懸念される場合は、影響のない時間帯にご実行ください。

  1. 以下の URL より “shacollector.zip” をダウンロードおよび展開いただき、対象端末へ配置します。

    ダウンロード先: shacollector.zip

    shacollector の使い方についてはこちら

    (注意) shacollector は機能の拡張などにより内容が変更されることがあります。 そのため、毎回上記のダウンロード先から shacollector を取得いただき利用してください。

  2. 管理者権限のコマンド プロンプトから配置した “shacollector” フォルダーへ移動して、shacollector を以下のように実行します。

    shacollector.bat support drive-space C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  3. 下記のメッセージが表示されますので、調査対象のドライブ名を入力後、Enterを押してください。

    Set drive name with one word, and push Enter key (C:\ > c) > c      ※Cドライブの場合

  4. 以下のコマンドを実行の上、サポート情報ログの採取を実施します。

    shacollector.bat support all C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  5. 生成されましたログの出力先フォルダーを圧縮の上、弊社までご提供ください。

以上です。
