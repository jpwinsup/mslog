一般的なネットワークのトラブルに関する調査に必要な資料について

本記事はマイクロソフト社員によって公開されております。

いつも弊社製品をご利用いただきまして誠にありがとうございます。
本サイトでは弊社にお問い合わせいただく際に、事前に採取しておいたほうが良い内容についてご紹介しております。
このページでは「一般的なネットワークのトラブルに関する調査に必要な資料について」紹介いたします。

本資料が有効なシナリオ

  • 接続元から接続先に対して、通信アプリケーションによって TCP や UDP 等の通信を行っても接続ができない
  • 接続元から接続先に対して、通信アプリケーションによって TCP や UDP 等の通信を行っている最中に突然不通となり切断される

など

採取いただく資料

  • パケット キャプチャー
  • TCP/IP 等の関連トレース
  • OS 構成情報
    • msinfo32.exe の結果
    • ipconfig の結果
  • イベント ログ

資料採取時の影響について

ログ採取中は各種リソースに多少の負荷がかかる可能性がございますが、通常実影響はございません。
また、OS 再起動の必要はございません。

対象機器

  • 接続元のコンピュータ
  • 接続先のコンピュータ

事前準備

トレース ログ格納用のフォルダーとして、接続元、ならびに接続先のコンピュータ上の C ドライブ配下に temp というフォルダー名でフォルダーを作成します。

資料採取手順

接続元、接続先のコンピュータ上で下記をご実施ください。

補足

ログ取得によるコマンドの実行は、個々のコマンドで実行されるトレースが最大サイズ 1024MB に達した場合には上書きします。
そのため、事象の再現を確認した後は、ログが上書きされないように速やかにトレース ログの停止をお願い申し上げます。
また情報採取にあたっては、管理権限が必要となりますことご留意ください。

a. 資料採取開始

操作対象:
接続元、接続先のコンピューターそれぞれで下記を実施ください。

  1. 管理者権限でコマンド プロンプトを起動します。
  2. 次のコマンドを実行します。

b. 事象再現

操作対象:
事象再現に必要なコンピューターで実施ください。

  1. 事象の再現に必要な操作を実施します。
  2. 事象が発生 (ネットワーク接続できない、また突然切断されるなど) したことを確認します。
    また、このときの日時を可能な限り正確に控えます。
    (※ 例：何時何分何秒に接続を開始し、何時何分何秒に事象が発生した等)

c. 資料採取停止

操作対象:
接続元、接続先のコンピューターそれぞれで下記を実施ください。

  1. 管理者権限でコマンド プロンプトを起動します。
  2. 次のコマンドを実行します。

d. その他の情報

  1. 管理者権限でコマンド プロンプトを起動します。
  2. 次のコマンドを実行します。

e. 資料提供

  1. 接続元、接続先のコンピュータ上で取得した C:\temp を 1 つのフォルダーにおまとめいただき zip 形式で圧縮します。
  2. 弊社まで事象を確認した日時、ならびにご取得いただいた資料をご提供ください。

本情報の内容（添付文書、リンク先などを含む）は、作成日時点でのものであり、予告なく変更される場合があります。
