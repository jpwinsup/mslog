DNS による名前解決に失敗する事象の調査に必要な資料について

いつも弊社製品をご利用いただきまして誠にありがとうございます。
本サイトでは弊社にお問い合わせいただく際に、事前に採取しておいたほうが良い内容についてご紹介しております。
このページでは「DNS による名前解決に失敗する事象の調査に必要な資料について」紹介いたします。

本資料が有効なシナリオ

  • DNS を利用した名前解決に失敗する

採取いただく資料

  • パケット キャプチャー
  • DNS クライアントおよびサーバー トレース ログ
  • OS 構成情報

資料採取時の影響について

ログ採取中は各種リソースに多少の負荷がかかる可能性がございますが、通常実影響はございません。
再起動の必要はございません。

対象機器

  • 名前解決に失敗する DNS クライアント
  • 上記クライアントが使用する DNS サーバー

資料採取手順

a. DNS サーバーで資料採取を開始

  1. DNS サーバーにログオンして情報出力用に C:\MSLOG フォルダーを作成します。
  2. コマンドプロンプトを管理者として起動します。
  3. 次のコマンドをそれぞれ実行して DNS の詳細トレース、パケットキャプチャの取得を開始します。
  4. Windows PowerShell を管理者として起動します。
  5. 次のコマンドを実行して DNS サーバーのキャッシュをクリアします。

b. DNS クライアントで資料採取の開始、事象の再現

  1. 不要な情報の出力を防ぐため、デスクトップ上のウィンドウを可能な限り閉じます。
  2. クライアント側でネットワークアダプターのプロパティから、問題が発生した DNS サーバーを優先 DNS サーバーとして設定します。
  3. 情報出力用に C:\MSLOG フォルダーを作成します。
  4. コマンドプロンプトを管理者として起動します。
  5. 次のコマンドをそれぞれ実行して DNS の詳細トレース、パケットキャプチャの取得を開始します。
  6. 次のコマンドをそれぞれ実行して各種キャッシュをクリアします。
  7. 次のコマンドを実行して、[ステップ記録ツール] を起動します。
    マウスをクリックするたびにマウスカーソル部分が赤く点滅し、画面スクリーンショットが取得されます。
  8. Windows PowerShell を起動します。
  9. Resolve-DnsName コマンドレットでの名前解決を試み、名前解決が失敗することを確認します。

c. DNS クライアントで資料採取の停止

  1. 次のコマンドを実行して、[ステップ記録ツール] を停止します。
  2. 次のコマンドを実行して、 DNS の詳細トレース、パケットキャプチャの取得を停止します。

d. DNS サーバーで資料採取の停止、構成情報の出力

  1. 次のコマンドを実行して、 DNS の詳細トレース、パケットキャプチャの取得を停止します。
  2. 次のコマンドをそれぞれ実行して環境情報を出力します。
    ※ 複数のコマンドをまとめてコピーして貼り付けることができます。環境によっては一部コマンドでエラーが表示されますが、無視して実行を続けてください。
  3. Windows PowerShell を管理者として起動して、次のコマンドを実行します。
    ※ 複数のコマンドをまとめてコピーして貼り付けることができます。コマンドを最後まで実行したら、プロンプトが戻るまでキーボードの ENTER キーを 2 回押下します
  4. コマンドプロンプトに戻り、次のコマンドを実行してフォルダー名を変更します。

e. DNS クライアントで構成情報の出力

  1. 起動したコマンドプロンプトで次のコマンドをそれぞれ実行します。
    ※ 複数のコマンドをまとめてコピーして貼り付けることができます。環境によっては一部コマンドでエラーが表示されますが、無視して実行を続けてください。

f. 資料提供

  1. C:\MSLOG_<コンピューター名> フォルダーに必要な資料が保存されていますので、zip 形式で圧縮します。
  2. 弊社まで資料をご提供ください。
