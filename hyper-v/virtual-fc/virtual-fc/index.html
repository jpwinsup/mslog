仮想 FC に関する調査に必要な資料について

目次

  • [A] 事象の再現性がある場合
  • [B] 過去に発生したエラーであり、再現性が無い場合

いつも弊社製品をご利用いただきまして誠にありがとうございます。

Windows プラットフォームサポートの石田です。

弊社にお問い合わせいただく際に、事前に採取しておいたほうが良い内容についてご紹介させていただきます。

今回のテーマは「仮想 FC に関する調査に必要な資料について」です。

仮想 FC に関する調査に必要な資料については以下の資料をご取得ください。

  • 採取いただく資料

    • storport のトレースログ ※ 再現性がある場合
    • Hyper-V のトレースログ ※ 再現性がある場合
    • Get-VMFibreChannelHba コマンドレットの情報
    • 情報採取スクリプト (shacollector) の get-virtualfc オプションのログ情報
    • 情報採取スクリプト (shacollector) の support オプションのログ情報

  • 対象機器

    • 問題が発生している仮想マシンの所有者の Hyper-V ホスト

[A] 事象の再現性がある場合

※ Hyper-V ホストにて採取します。

  1. 以下の URL より “shacollector.zip” をダウンロードおよび展開いただき、対象端末へ配置します。

    ダウンロード先: shacollector.zip

    shacollector の使い方についてはこちら

    (注意) shacollector は機能の拡張などにより内容が変更されることがあります。 そのため、毎回上記のダウンロード先から shacollector を取得いただき利用してください。

  2. コマンドプロンプトを管理者モードで起動し、shacollector.zip を展開したフォルダーのバッチファイル (shacollector.bat) を以下のオプションを指定して実行します。

    shacollector.bat trace storport start size 4096 C:\mslog
shacollector.bat trace hyper-v start size 4096 C:\mslog

    ※ 最大ファイルサイズは 4096 以外のサイズ (MB) を size オプションに指定することも可能です。

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  3. 事象を再現させます。

    ※ ディスクが認識しない事象の場合は、ディスクの管理画面より再スキャンを行います。

  4. 以下のコマンドを実行して資料の採取を停止します。

    shacollector.bat trace hyper-v stop
shacollector.bat trace storport stop

  5. PowerShell を管理者モードで起動して以下のコマンドを実行します。

    Get-VMFibreChannelHba -VMName "仮想マシン名" > 出力ログファイル名

    ※ “仮想マシン名” は Hyper-V マネージャーで表示されている仮想マシンの名前をご指定ください。

    ※ また、仮想マシン毎に正常なものと異常なものがある場合は、比較のため正常な仮想マシンに対しての情報もご取得ください。

  6. バッチファイル (shacollector.bat) のログ情報を採取します。

    shacollector.bat support virtualfc C:\mslog
shacollector.bat support all C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

    ※ バッチファイルではログファイルのコピーやコマンドを実行しております。 システムに負荷をかけるような操作は実行しておりませんが、サービスへの 影響が懸念される場合は、影響のない時間帯にご実行ください。

  7. 生成されましたログの出力先フォルダーを圧縮の上、弊社までご提供ください。
