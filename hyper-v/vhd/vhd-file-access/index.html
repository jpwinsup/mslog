VHD ファイルアクセスが失敗する事象の調査

目次

  • [A] 事象の再現性がある場合
  • [B] 過去に発生したエラーであり、再現性が無い場合

これまで弊社サポートのトラブルシューティングに活用されていたログ採取ツール “Shacollector” がリニューアルされます。2023 年 9 月 から、TSS.ps1 を用いた手順に順次変更されて、より包括的なログ採取が可能になります。

いつも弊社製品をご利用いただきまして誠にありがとうございます。

Windows プラットフォームサポートの石田です。

弊社にお問い合わせいただく際に、事前に採取しておいたほうが良い内容についてご紹介させていただきます。

今回のテーマは「VHD ファイルアクセスが失敗する事象の調査に必要な資料について」です。

VHD ファイルアクセスが失敗する事象の調査に必要な資料については以下の資料をご取得ください。

  • 採取いただく資料

    • vhd のトレースログ ※ 再現性がある場合
    • プロセスモニターログ ※ 再現性がある場合
    • ストレージ関連のトレースログ ※ 再現性がある場合且つローカルディスクの VHD ファイルにアクセスする場合
    • SMB 関連のトレースログ ※ 再現性がある場合且つファイルサーバー上の VHD ファイルにアクセスする場合
    • 情報採取スクリプト (shacollector) の support オプションのログ情報

  • 対象機器

    • 問題が発生している仮想マシン

[A] 事象の再現性がある場合

  1. 以下の URL より “shacollector.zip” をダウンロードおよび展開いただき、対象端末へ配置します。

    ダウンロード先: shacollector.zip

    shacollector の使い方についてはこちら

    (注意) shacollector は機能の拡張などにより内容が変更されることがあります。 そのため、毎回上記のダウンロード先から shacollector を取得いただき利用してください。

  2. コマンドプロンプトを管理者モードで起動し、shacollector.zip を展開したフォルダーのバッチファイル (shacollector.bat) を以下のオプションを指定して実行します。

    shacollector.bat trace vhd start size 4096 C:\mslog
shacollector.bat trace storage start size 4096 C:\mslog　 ※ ローカルディスクの VHD ファイルにアクセスする場合に実施。
shacollector.bat trace procmon start C:\mslog

    ※ 最大ファイルサイズは 4096 以外のサイズ (MB) を size オプションに指定することも可能です。 ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。 ※ ファイルサーバー上の VHD にアクセスする場合は、ファイル共有 (SMB 通信) に関する一般的な事象の調査に必要な資料について も同時に採取します。 ※ ファイルサーバーの資料採取でもプロセスモニターのログ情報の案内がございますが、クライアント側での採取はこちらで行っておりますので、サーバー側のみの採取をお願いいたします。

  3. VHD ファイルにアクセスして事象を再現させます。

  4. 以下のコマンドを実行して資料の採取を停止します。

    shacollector.bat trace procmon stop
shacollector.bat trace storage stop     ※ ローカルディスクの VHD ファイルにアクセスする場合に実施。
shacollector.bat trace vhd stop

  5. バッチファイル (shacollector.bat) のログ情報を採取します。

    shacollector.bat support all C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

    ※ バッチファイルではログファイルのコピーやコマンドを実行しております。 システムに負荷をかけるような操作は実行しておりませんが、サービスへの 影響が懸念される場合は、影響のない時間帯にご実行ください。

  6. 生成されましたログの出力先フォルダーを圧縮の上、弊社までご提供ください。
