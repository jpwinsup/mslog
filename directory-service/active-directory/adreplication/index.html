このページでは「ドメイン コントローラー複製障害の調査」に必要な資料採取についてご紹介いたします。

本資料が有効なシナリオ

  • ドメイン コントローラー間の複製が失敗する。(repadmin /showrepl の結果でエラーが発生している状況)

採取いただく資料

  • パケット キャプチャー
  • OS 構成情報
  • イベント ログ
  • グループ ポリシー関連情報
  • 問題ステップ記録ツール

など

資料採取時の影響について

ログ採取中は各種リソースに多少の負荷がかかる可能性がございますが、通常実影響はございません。
また、再起動の必要はございません。

対象機器

  • ドメイン コントローラー (複製エラーの発生しているドメイン コントローラーと複製パートナーのドメイン コントローラー) の両方

事前準備 および 情報採取手順

採取対象機器にて、以下のリンクを参考に情報採取を実施していただきますようお願いいたします。

TSS 情報採取手順

なお、1 の手順内の [2] 事象再現 にあたっては、複製エラーの事象を再現させることを目的とし、
複製エラーの発生しているドメイン コントローラーにて、管理者権限のコマンドプロンプトを開き、以下のコマンドを実行します。

repadmin /syncall /Ae && repadmin /syncall /APe

複製エラーが発生したことを確認して、リンク先の [3] TSS.ps1 ログの採取停止 にお進みください。
また、その際、エラーが発生した日時を併せてお知らせください。

手順は以上でございます。
