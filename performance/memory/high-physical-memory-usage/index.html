物理メモリの使用率が高い事象の調査

目次

  • 資料採取の流れ
  • ログ採取について

本記事はマイクロソフト社員によって公開されております。

いつも弊社製品をご利用いただきまして誠にありがとうございます。Windows プラットフォーム サポートの栗木です。
本記事では、物理メモリの使用率が高い事象が発生した場合に採取いただきたい資料についてご紹介いたします。

資料採取の流れ

  1. RAMMap ツール と TSS ツール を資料採取対象のマシンの任意の場所にコピーし展開します。
  2. RAMMap ツール を実行し、実行結果を保存します。
  3. TSS ツールを利用してサポートログ情報を採取します。

ログ採取について

  1. RAMMap.zip と TSS.zip をダウンロードおよび展開いただき、対象端末へ配置します。

    (注意)
    これらのツールは機能の拡張などにより内容が変更されることがあります。
    そのため、毎回上記のダウンロード先から取得していただき利用してください。

  2. RAMMap フォルダーにある RAMMap.exe を実行し、次のような画面が表示されるまで少しお待ちください。

    表示されましたら左上の File から Save をクリックしていただきデータを RMP ファイルとして保存します。

    保存した RMP ファイルを圧縮の上、弊社までご提供ください。

    (注意)
    お客様の環境によっては、稀にRMP ファイルの保存に失敗する事例が報告されております。
    そのため、ご提供いただく前に保存した RMP ファイルをダブルクリックして開き、正常にファイルが開かれることを確認してください。

  3. 管理者権限で起動した PowerShell で配置した TSS フォルダーへ移動後、以下のコマンドを実行し、サポート情報ログを採取します。

    Set-ExecutionPolicy bypass -Scope Process -Force  
.\TSS.ps1 -CollectLog SHA_support-all -noBasicLog -noUpdate

    ログファイルは既定で C:\MS_DATA に出力されます。別の場所に出力されたい場合には -LogFolderPath オプションを付与してください。

    実行例：.\TSS.ps1 -CollectLog SHA_support-all -noBasicLog -noUpdate -LogFolderPath D:\MS_DATA

  4. 生成されましたサポート情報ログの出力先フォルダーを圧縮の上、弊社までご提供ください。

以上となります。 本ブログが少しでも皆様のお役に立てば幸いです。
