プロセスのメモリ使用量が多い事象の調査

目次

  • 資料採取の流れ
  • ログ採取について

本記事はマイクロソフト社員によって公開されております。

いつも弊社製品をご利用いただきまして誠にありがとうございます。Windows プラットフォーム サポートの栗木です。 本記事では、プロセスのメモリ使用量が多い事象が発生した場合に採取いただきたい資料についてご紹介いたします。

資料採取の流れ

  1. VMMap ツール と shacollector ツール を資料採取対象のマシンの任意の場所にコピーし展開します。
  2. VMMap ツール を実行し、実行結果を保存します。
  3. shacollector ツールを利用してサポートログ情報を採取します。

ログ採取について

  1. 以下の URL より VMMap.zip と shacollector.zip をダウンロードおよび展開いただき、対象端末へ配置します。

    VMMap: https://download.sysinternals.com/files/VMMap.zip shacollector: shacollector.zip

    (注意) これらのツールは機能の拡張などにより内容が変更されることがあります。 そのため、毎回上記のダウンロード先から取得していただき利用してください。

  2. VMMap フォルダーにある VMMap64.exe を管理者権限で実行します。

    次のような画面が表示されるので調査対象であるメモリ使用量の多いプロセスを選択してください。

    次のようなメモリ使用状況が表示されるまで少しお待ちください。

    表示されましたら左上の File から Save をクリックしていただきデータを mmp ファイルとして保存します。

    保存した mmp ファイルを圧縮の上、弊社までご提供ください。 (注意) 管理者権限でなくとも VMMap は、実行可能ですが、確認できるプロセスに制限があるため、管理者権限で実行してください。

  3. 管理者権限のコマンド プロンプトから配置した shacollector フォルダーへ移動後、以下のコマンドを実行し、サポート情報ログを採取します。

    shacollector.bat support all C:\mslog

    ※ 出力先フォルダー名に C:\mslog 以外のフォルダーを指定することも可能です。

  4. 生成されましたサポート情報ログの出力先フォルダーを圧縮の上、弊社までご提供ください。

以上となります。 本ブログが少しでも皆様のお役に立てば幸いです。
