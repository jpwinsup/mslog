DFS 名前空間パスへのアクセスに関する一般的な事象の調査に必要な資料について

  • 対象端末
  • 採取手順
    • 注意事項

本記事はマイクロソフト社員によって公開されております。

いつも弊社製品をご利用いただきまして誠にありがとうございます。
本サイトでは弊社にお問い合わせいただく際に、事前に採取しておいたほうが良い内容についてご紹介しております。
このページでは「DFS 名前空間パスへのアクセスに関する一般的な事象の調査に必要な資料について」紹介いたします。

本資料が有効なシナリオ

  • DFS 名前空間を利用したファイル共有に接続できない
  • DFS 名前空間を利用したファイル共有上のファイル操作に失敗する

など

対象端末

  • クライアント端末
  • DFS 名前空間サーバー
  • ドメイン コントローラー

DFS 名前空間にアクセスできない場合、被疑箇所がドメイン コントローラー、名前空間サーバー、ファイル サーバー、クライアントなど多数あり、かついずれのサーバーも複数存在することが多いため、全台での情報採取は現実的でない場合があろうかと存じます。

そのため、まずクライアント側のみで正常時と異常時の情報を採取して被疑箇所を絞り込んでいく方法もございます。

採取手順

  1. 以下の URL から TSS.zip をダウンロードし、Zip を展開します。

    https://aka.ms/getTSS

  2. 展開した TSS フォルダーを調査対象となる端末上の任意のパスに配置します。

  3. Windows PowerShell (powershell.exe) を [管理者として実行] にて開始します。

  4. “cd” コマンドで TSS フォルダーを配置したパスにカレント フォルダーを移動します。 例として、“C:\Test” フォルダーに TSS フォルダーを配置した場合は以下のように実行します。

    cd C:\Test\TSS

  5. 以下のコマンドでログ取得を開始します。

    # クライアント側で実行するコマンド
Set-ExecutionPolicy RemoteSigned -Scope Process -Force
.\TSS.ps1 -Scenario NET_DFScli

    # サーバー側で実行するコマンド
Set-ExecutionPolicy RemoteSigned -Scope Process -Force
.\TSS.ps1 -Scenario NET_DFSsrv
    • 使用許諾の確認 (End User License Agreement) が表示された場合は内容を読んでいただき、[Accept] ボタンで同意していただきますようお願い申し上げます。
    • ステップ記録ツール (Problem Step Recording : PSR) に関する確認が表示された場合は [Y] キーで同意していただきますようお願い申し上げます。
    [Action-Privacy] We need your consent to allow Problem Step Recording and-or Screen-Video recording, please answer Y or N
Press Y for Yes = allow recording, N for No [Y,N]?

  6. 下のメッセージが表示された状態で止まったことを確認し、次の手順へ進みます。

    Reproduce the issue and enter 'Y' key AFTER finishing the repro (with window focus here) [Y]?

    クライアントおよびサーバーの両方で上記のメッセージが表示されたことを確認し、次の手順に進んでください。

  7. エクスプローラーを起動して DFS 名前空間の UNC パスにアクセスします。

  8. 手順 6. のメッセージに対して [Y] キーを入力し、ログ取得を停止します。

    サーバー側のみ、下のメッセージが表示された状態で止まったら [Y] キーを入力します。

    Press Y for Yes = collect DFSutil, N for No (timeout=30s) [Y,N]?

    その後、下のメッセージが表示された状態で止まったら、<調査対象の名前空間> を入力して [Enter] を入力します。

    Please specify DFS root, i.e. \\contoso.com\your_DFSroot_name - no subfolders, or hit ENTER to skip this step:

  9. 各種ログの取得処理が行われますので、処理完了までお待ちください。

  10. C:\MS_DATA\ フォルダーを zip 形式に圧縮し、アップロード サイトからご提供ください。

注意事項

もし情報採取中に何らかのエラーが発生した場合は、以下の対応を実施ください。

  • PowerShell の実行画面からエラー内容が確認できるスクリーン ショットもしくは、エラー内容をコピーした情報をお寄せください。
  • 実行状況の確認のため .\TSS.ps1 -Status コマンドの実行結果をお知らせください。
  • .\TSS.ps1 -Stop コマンドを実行し、情報採取を停止ください。
